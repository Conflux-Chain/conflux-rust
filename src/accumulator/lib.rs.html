10//! This module provides algorithms for accessing and updating a Merkle
11//! Accumulator structure persisted in a key-value store. Note that this doesn't
12//! write to the storage directly, rather, it reads from it via the `HashReader`
13//! trait and yields writes via an in memory `HashMap`.
14//!
15//! # Merkle Accumulator
16//! Given an ever growing (append only) series of "leaf" hashes, we construct an
17//! evolving Merkle Tree for which proofs of inclusion/exclusion of a leaf hash
18//! at a leaf index in a snapshot of the tree (represented by root hash) can be
19//! given.
20//!
21//! # Leaf Nodes
22//! Leaf nodes carry hash values to be stored and proved. They are only appended
23//! to the tree but never deleted or updated.
24//!
25//! # Internal Nodes
26//! A non-leaf node carries the hash value derived from both its left and right
27//! children.
28//!
29//! # Placeholder Nodes
30//! To make sure each Leaf node has a Merkle Proof towards the root, placeholder
31//! nodes are added so that along the route from a leaf to the root, each node
32//! has a sibling. Placeholder nodes have the hash value
33//! `ACCUMULATOR_PLACEHOLDER_HASH`
34//!
35//! A placeholder node can appear as either a Leaf node or a non-Leaf node, but
36//! there is at most one placeholder leaf at any time.
37//!
38//! # Frozen Nodes & Non-frozen Nodes
39//! As leaves are added to the tree, placeholder nodes get replaced by
40//! non-placeholder nodes, and when a node has all its descendants being
41//! non-placeholder, it becomes "Frozen" -- its hash value won't change again in
42//! the event of new leaves being added. All leaves appended (not counting the
43//! one possible placeholder leaf) are by definition Frozen.
44//!
45//! Other nodes, which have one or more placeholder descendants are Non-Frozen.
46//! As new elements are appended to the accumulator the hash value of these
47//! nodes will change.
48//!
49//! # Leaf Count
50//! Given a count of the number of leaves in a Merkle Accumulator it is possible
51//! to determine the shape of the accumulator -- which nodes are filled and
52//! which nodes are placeholder nodes.
53//!
54//! Example:
55//! Logical view of a Merkle Accumulator with 5 leaves:
56//! ```text
57//!            Non-fzn
58//!           /       \
59//!          /         \
60//!         /           \
61//!       Fzn2         Non-fzn
62//!      /   \           /   \
63//!     /     \         /     \
64//!    Fzn1    Fzn3  Non-fzn  [Placeholder]
65//!   /  \    /  \    /    \
66//!  L0  L1  L2  L3 L4   [Placeholder]
67//! ```
68//!
69//! # Position and Physical Representation
70//! As a Merkle Accumulator tree expands to the right and upwards, we number
71//! newly frozen nodes monotonically. One way to do it is simply to use in-order
72//! index of nodes, and this is what we do for the in-memory representation. We
73//! call the stated numbers identifying nodes below simply "Position", and
74//! unless otherwise stated, this is the in-order position.
75//!
76//! For writing to disk however, we write all the children of a node before the
77//! parent. Thus for disk write order, it is more convenient to use the
78//! post-order position as an index. And with that we can map a Merkle
79//! Accumulator into a key-value storage: key is the post-order position of a
80//! node, and the value is hash value it carries.
81//!
82//! We store only Frozen nodes, and generate non-Frozen nodes on the fly when
83//! accessing the tree. This way, the physical representation of the tree is
84//! append-only, i.e. once written to physical storage, nodes won't be either
85//! modified or deleted.
86//!
87//! Here is what we persist for the logical tree in the above example:
88//!
89//! ```text
90//!          Fzn2(6)
91//!         /      \
92//!        /        \
93//!    Fzn1(2)       Fzn3(5)
94//!   /     \       /     \
95//!  L0(0)  L1(1)  L2(3)  L3(4)  L4(7)
96//! ```
97//!
98//! When the next leaf node is persisted, the physical representation will be:
99//!
100//! ```text
101//!          Fzn2(6)
102//!         /      \
103//!        /        \
104//!    Fzn1(2)       Fzn3(5)       Fzn4(9)
105//!   /     \       /     \       /      \
106//!  L0(0)  L1(1)  L2(3)  L3(4)  L4(7)   L5(8)
107//! ```
108//!
109//! The numbering corresponds to the post-order traversal of the tree.
110//!
111//! To think in key-value pairs:
112//! ```text
113//! |<-key->|<--value-->|
114//! |   0   | hash_L0   |
115//! |   1   | hash_L1   |
116//! |   2   | hash_Fzn1 |
117//! |  ...  |   ...     |
118//! ```

136/// Defines the interface between `MerkleAccumulator` and underlying storage.
137pub trait HashReader {
138    /// Return `HashValue` carried by the node at `Position`.
139    fn get(&self, position: Position) -> Result<HashValue>;
140}

142/// A `Node` in a `MerkleAccumulator` tree is a `HashValue` at a `Position`
143type Node = (Position, HashValue);

156    /// Given an existing Merkle Accumulator (represented by
157    /// `num_existing_leaves` and a `reader` that is able to fetch all
158    /// existing frozen nodes), and a list of leaves to be appended, returns
159    /// the result root hash and new nodes to be frozen.
160    pub fn append(
161        reader: &R, num_existing_leaves: LeafCount, new_leaves: &[HashValue],
162    ) -> Result<(HashValue, Vec<Node>)> {
163        MerkleAccumulatorView::<R, H>::new(reader, num_existing_leaves)
164            .append(new_leaves)
165    }

242/// Actual implementation of Merkle Accumulator algorithms, which carries the
243/// `reader` and `num_leaves` on an instance for convenience
244struct MerkleAccumulatorView<'a, R, H> {
245    reader: &'a R,
246    num_leaves: LeafCount,
247    hasher: PhantomData<H>,
248}
249
250impl<'a, R, H> MerkleAccumulatorView<'a, R, H>
251where
252    R: HashReader,
253    H: CryptoHasher,
254{
255    fn new(reader: &'a R, num_leaves: LeafCount) -> Self {
256        Self {
257            reader,
258            num_leaves,
259            hasher: PhantomData,
260        }
261    }
262
263    /// implementation for pub interface `MerkleAccumulator::append`
264    fn append(
265        &self, new_leaves: &[HashValue],
266    ) -> Result<(HashValue, Vec<Node>)> {
267        // Deal with the case where new_leaves is empty
268        if new_leaves.is_empty() {
269            if self.num_leaves == 0 {
270                return Ok((*ACCUMULATOR_PLACEHOLDER_HASH, Vec::new()));
271            } else {
272                let root_hash = self.get_hash(
273                    Position::root_from_leaf_count(self.num_leaves),
274                )?;
275                return Ok((root_hash, Vec::new()));
276            }
277        }
278
279        let num_new_leaves = new_leaves.len();
280        let last_new_leaf_count = self.num_leaves + num_new_leaves as LeafCount;
281        let root_level =
282            Position::root_level_from_leaf_count(last_new_leaf_count);
283        let mut to_freeze =
284            Vec::with_capacity(Self::max_to_freeze(num_new_leaves, root_level));
285
286        // Iterate over the new leaves, adding them to to_freeze and then adding
287        // any frozen parents when right children are encountered.  This
288        // has the effect of creating frozen nodes in
289        // perfect post-order, which can be used as a strictly increasing append
290        // only index for the underlying storage.
291        //
292        // We will track newly created left siblings while iterating so we can
293        // pair them with their right sibling, if and when it becomes
294        // frozen.  If the frozen left sibling is not created
295        // in this iteration, it must already exist in storage.
296        let mut left_siblings: Vec<(_, _)> = Vec::new();
297        for (leaf_offset, leaf) in new_leaves.iter().enumerate() {
298            let leaf_pos = Position::from_leaf_index(
299                self.num_leaves + leaf_offset as LeafCount,
300            );
301            let mut hash = *leaf;
302            to_freeze.push((leaf_pos, hash));
303            let mut pos = leaf_pos;
304            while pos.is_right_child() {
305                let sibling = pos.sibling();
306                hash = match left_siblings.pop() {
307                    Some((x, left_hash)) => {
308                        assert_eq!(x, sibling);
309                        Self::hash_internal_node(left_hash, hash)
310                    }
311                    None => Self::hash_internal_node(
312                        self.reader.get(sibling)?,
313                        hash,
314                    ),
315                };
316                pos = pos.parent();
317                to_freeze.push((pos, hash));
318            }
319            // The node remaining must be a left child, possibly the root of a
320            // complete binary tree.
321            left_siblings.push((pos, hash));
322        }
323
324        // Now reconstruct the final root hash by walking up to root level and
325        // adding placeholder hash nodes as needed on the right, and
326        // left siblings that have either been newly created or read
327        // from storage.
328        let (mut pos, mut hash) =
329            left_siblings.pop().expect("Must have at least one node");
330        for _ in pos.level()..root_level as u32 {
331            hash = if pos.is_left_child() {
332                Self::hash_internal_node(hash, *ACCUMULATOR_PLACEHOLDER_HASH)
333            } else {
334                let sibling = pos.sibling();
335                match left_siblings.pop() {
336                    Some((x, left_hash)) => {
337                        assert_eq!(x, sibling);
338                        Self::hash_internal_node(left_hash, hash)
339                    }
340                    None => Self::hash_internal_node(
341                        self.reader.get(sibling)?,
342                        hash,
343                    ),
344                }
345            };
346            pos = pos.parent();
347        }
348        assert!(left_siblings.is_empty());
349
350        Ok((hash, to_freeze))
351    }
352
353    /// upper bound of num of frozen nodes:
354    ///     new leaves and resulting frozen internal nodes forming a complete
355    /// binary subtree         num_new_leaves * 2 - 1 < num_new_leaves * 2
356    ///     and the full route from root of that subtree to the accumulator root
357    /// turns frozen         height - (log2(num_new_leaves) + 1) < height -
358    /// 1 = root_level
359    fn max_to_freeze(num_new_leaves: usize, root_level: u32) -> usize {
360        precondition!(root_level as usize <= MAX_ACCUMULATOR_PROOF_DEPTH);
361        precondition!(num_new_leaves < (usize::max_value() / 2));
362        precondition!(
363            num_new_leaves * 2 <= usize::max_value() - root_level as usize
364        );
365        num_new_leaves * 2 + root_level as usize
366    }
367
368    fn hash_internal_node(left: HashValue, right: HashValue) -> HashValue {
369        MerkleTreeInternalNode::<H>::new(left, right).hash()
370    }
371
372    fn rightmost_leaf_index(&self) -> u64 { (self.num_leaves - 1) as u64 }
373
374    fn get_hash(&self, position: Position) -> Result<HashValue> {
375        let idx = self.rightmost_leaf_index();
376        if position.is_placeholder(idx) {
377            Ok(*ACCUMULATOR_PLACEHOLDER_HASH)
378        } else if position.is_freezable(idx) {
379            self.reader.get(position)
380        } else {
381            // non-frozen non-placeholder node
382            Ok(Self::hash_internal_node(
383                self.get_hash(position.left_child())?,
384                self.get_hash(position.right_child())?,
385            ))
386        }
387    }
388
389    fn get_hashes(&self, positions: &[Position]) -> Result<Vec<HashValue>> {
390        positions.iter().map(|p| self.get_hash(*p)).collect()
391    }
392
393    fn get_root_hash(&self) -> Result<HashValue> {
394        self.get_hash(Position::root_from_leaf_count(self.num_leaves))
395    }
396
397    /// implementation for pub interface `MerkleAccumulator::get_proof`
398    fn get_proof(&self, leaf_index: u64) -> Result<AccumulatorProof<H>> {
399        ensure!(
400            leaf_index < self.num_leaves as u64,
401            "invalid leaf_index {}, num_leaves {}",
402            leaf_index,
403            self.num_leaves
404        );
405
406        let siblings = self.get_siblings(leaf_index, |_p| true)?;
407        Ok(AccumulatorProof::new(siblings))
408    }
409
410    /// Implementation for public interface
411    /// `MerkleAccumulator::get_consistency_proof`.
412    fn get_consistency_proof(
413        &self, sub_acc_leaves: LeafCount,
414    ) -> Result<AccumulatorConsistencyProof> {
415        ensure!(
416            sub_acc_leaves <= self.num_leaves,
417            "Can't get accumulator consistency proof for a version newer than the local version. \
418            Local next version: {}, asked next version: {}",
419            self.num_leaves,
420            sub_acc_leaves,
421        );
422
423        let subtrees =
424            FrozenSubtreeSiblingIterator::new(sub_acc_leaves, self.num_leaves)
425                .map(|p| self.reader.get(p))
426                .collect::<Result<Vec<_>>>()?;
427
428        Ok(AccumulatorConsistencyProof::new(subtrees))
429    }
430
431    /// Implementation for public interface
432    /// `MerkleAccumulator::get_range_proof`.
433    fn get_range_proof(
434        &self, first_leaf_index: Option<u64>, num_leaves: LeafCount,
435    ) -> Result<AccumulatorRangeProof<H>> {
436        let (left_siblings, right_siblings) =
437            self.get_range_proof_positions(first_leaf_index, num_leaves)?;
438        Ok(AccumulatorRangeProof::new(
439            self.get_hashes(&left_siblings)?,
440            self.get_hashes(&right_siblings)?,
441        ))
442    }
443
444    fn get_range_proof_positions(
445        &self, first_leaf_index: Option<u64>, num_leaves: LeafCount,
446    ) -> Result<(Vec<Position>, Vec<Position>)> {
447        if first_leaf_index.is_none() {
448            ensure!(
449                num_leaves == 0,
450                "num_leaves is not zero while first_leaf_index is None.",
451            );
452            return Ok((Vec::new(), Vec::new()));
453        }
454
455        let first_leaf_index =
456            first_leaf_index.expect("first_leaf_index should not be None.");
457        ensure!(
458            num_leaves > 0,
459            "num_leaves is zero while first_leaf_index is not None.",
460        );
461        let last_leaf_index = first_leaf_index
462            .checked_add(num_leaves - 1)
463            .ok_or_else(|| format_err!("Requesting too many leaves."))?;
464        ensure!(
465            last_leaf_index < self.num_leaves as u64,
466            "Invalid last_leaf_index: {}, num_leaves: {}",
467            last_leaf_index,
468            self.num_leaves,
469        );
470
471        let left_siblings =
472            self.get_sibling_positions(first_leaf_index, |p| p.is_left_child());
473        let right_siblings =
474            self.get_sibling_positions(last_leaf_index, |p| p.is_right_child());
475        Ok((left_siblings, right_siblings))
476    }
477
478    fn get_siblings(
479        &self, leaf_index: u64, filter: impl Fn(Position) -> bool,
480    ) -> Result<Vec<HashValue>> {
481        self.get_hashes(&self.get_sibling_positions(leaf_index, filter))
482    }
483
484    /// Helper function to get siblings on the path from the given leaf to the
485    /// root. An additional filter function can be applied to filter out
486    /// certain siblings.
487    fn get_sibling_positions(
488        &self, leaf_index: u64, filter: impl Fn(Position) -> bool,
489    ) -> Vec<Position> {
490        let root_pos = Position::root_from_leaf_count(self.num_leaves);
491        Position::from_leaf_index(leaf_index)
492            .iter_ancestor_sibling()
493            .take(root_pos.level() as usize)
494            .filter(|p| filter(*p))
495            .collect()
496    }
497
498    /// Implementation for public interface
499    /// `MerkleAccumulator::get_frozen_subtree_hashes`.
500    fn get_frozen_subtree_hashes(&self) -> Result<Vec<HashValue>> {
501        FrozenSubTreeIterator::new(self.num_leaves)
502            .map(|p| self.reader.get(p))
503            .collect::<Result<Vec<_>>>()
504    }
505}
